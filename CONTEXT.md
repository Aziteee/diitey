# Diitey

Diitey 是一个以文件为内容真相、由主题解释网站结构、由插件提供动态能力的个人站点系统。核心协调三者，并保证发布视图的一致性。

## 内容

**内容文件**：
位于站点内容目录中的 Markdown 源文件，是文章正文与属性的唯一真相来源。
_避免使用_：文章数据、内容条目、数据库文章

**内容 ID**：
内容文件声明的全站唯一稳定身份；内容移动、改名或更换主题时保持不变。
_避免使用_：文章 ID、文件 ID、slug

**内容记录**：
核心从一个内容文件得到的结构化发布表示，保留内容身份、来源、属性与正文结果。
_避免使用_：文章对象、数据库记录

**内容快照**：
一次完整构建产生的、内部一致且只读的全部内容记录与索引；不包含页面 HTML、插件动态数据或 island 资源。
_避免使用_：内容缓存、索引缓存、发布结果

**有效发布视图**：
站点程序与某份内容快照组合后、可原子服务的整体发布结果；包含路由表、静态页面、请求期页面计划、island 资源与主题样式表资源。任一请求从开始到结束只使用一份有效发布视图。
_避免使用_：内容缓存、页面缓存、最新快照

**有效快照**：
历史用语，现指有效发布视图所依赖的那份成功内容快照；构建失败不会改变当前有效发布视图。
_避免使用_：最新快照、当前缓存

## 信息架构

**主题**：
站点信息架构与呈现方式的所有者，定义集合、路由、页面和浏览器交互组件；主题不拥有内容。
_避免使用_：模板、皮肤、前端插件

**集合**：
主题声明的内容记录查询，用于选择、过滤和排序内容；集合不复制或拥有其结果。
_避免使用_：栏目、内容表、文章目录

**路由映射**：
主题声明的 URL 模式与内容源路径模式之间的确定关系。
_避免使用_：文件路由、重定向规则

**canonical URL**：
一条内容记录被多个路由发布时，由主题明确指定并由内容链接采用的主要 URL。
_避免使用_：原始 URL、永久 URL

**island**：
嵌入服务端页面并在浏览器中独立获得交互能力的组件。主题拥有发布 island；插件可在核心 admin surface 内声明插件 admin island（见 admin surface）。
_避免使用_：页面脚本、小部件

**admin surface**：
核心拥有的受鉴权管理界面，挂在 publicServer 的 `/_admin` 命名空间；包含登录会话、document、资源托管与 admin Action 分派，不属于主题发布信息架构。
_避免使用_：CMS 后台、主题管理页、插件后台路由

**插件 admin 页**：
插件在 admin surface 内声明的至多一个管理页面；路径由 plugin ID 推导，浏览器组件由核心构建为插件 admin island。
_避免使用_：插件路由、自定义后台路径

**文档壳 / document**：
主题可选声明的全站页面外壳组件，渲染完整文档树（`<html>` / `<head>` / `<body>`）与共享 chrome，并包裹各路由页面；不是内容文件或内容记录。未声明时核心使用最小回退壳。
_避免使用_：文档、内容文档、HTML 模板、布局插件

**主题样式表**：
主题可选声明、由核心在站点程序编译时构建并哈希托管的 CSS 资源（`/assets/theme/*`）；主题选择 CSS 工具链（含可选 Tailwind），核心只负责构建、钉住与提供 URL。
_避免使用_：全局样式插件、核心 Tailwind、内容快照样式

## 动态能力

**插件**：
为站点提供服务端动态能力或静态 Markdown 扩展的本地扩展；插件不拥有发布页面布局、主题路由或发布 islands，可在核心 admin surface 内声明至多一个管理页。
_避免使用_：主题扩展、前端插件

**插件服务**：
插件声明、仅在服务端调用的类型化能力，是主题与插件动态数据交互的受支持边界。
_避免使用_：插件 API、插件方法、数据表接口

**Action**：
浏览器经核心统一入口调用插件服务的受控操作。public Action 使用 `/_action/<name>`；admin Action 使用 `/_admin/action/<plugin-id>/<name>`，并要求 admin 会话。
_避免使用_：插件路由、公开服务、任意接口

**Markdown 扩展**：
插件声明的内容转换规则，把自定义 Markdown 语法变成静态内容片段；其视觉表现仍由主题决定。
_避免使用_：Markdown 组件、插件 island

**动态数据**：
评论等运行期间产生、由插件拥有的数据；不包含内容文件的正文与属性。
_避免使用_：内容数据、文章数据

**插件迁移**：
插件声明、由核心在服务启动前自动应用的可追踪动态数据结构版本变更。
_避免使用_：自动建表

**扩展配置**：
站点所有者为所选主题或插件提供、在站点程序形成时固定的参数；不属于动态数据，变更需要形成新的站点程序。
_避免使用_：运行时设置、插件数据、Data 配置

## 运行

**站点程序**：
服务启动时固定的主题、插件、页面计划、island 构建结果、主题样式表构建结果与程序版本；内容 reload 只能使用这份程序，主题、插件或 `site.config.ts` 的变化须通过进程重启生效。
_避免使用_：主题缓存、扩展运行时、热加载扩展

**reload**：
站点所有者主动发起的内容重建；成功时用新的内容快照生成并原子替换有效发布视图，失败时保留原有效发布视图。不重载主题、插件、islands、主题样式表，也不执行插件迁移。
_避免使用_：热更新、自动同步、重启

**发布运行时**：
核心对外提供的发布协调边界，负责请求处理、内容 reload、状态查询与资源关闭。
_避免使用_：站点服务器、主题引擎

**核心**：
协调内容、主题与插件并维护受支持边界的站点运行主体；核心不拥有内容结构或插件动态数据。
_避免使用_：CMS 后台、主题引擎
