# 插件限于服务端能力与静态 Markdown 扩展

插件只通过类型化插件服务、核心 Action 和静态 Markdown 扩展提供发布侧能力，不注入页面布局、主题路由、发布页面浏览器组件或主题 islands。相比让插件携带完整前后端 UI，这个边界避免主题与插件争夺呈现控制权，并保持发布浏览器资源由主题统一构建；需要交互的发布功能由主题 island 调用 public Action，静态扩展只产生由主题负责样式的内容标记。

**例外**：插件可在核心拥有的受鉴权 admin surface 内声明至多一个浏览器组件（插件 admin island），见 ADR-0013。该例外不授权插件注入发布页布局、主题路由或主题 islands。
