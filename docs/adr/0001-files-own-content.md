# 内容由文件拥有

Diitey 将内容目录中的 Markdown 文件作为正文与属性的唯一真相来源，SQLite 只保存插件动态数据，不复制文章内容。相比以数据库为中心的 CMS 或同时维护文件与数据库副本，这保留了 Obsidian、Git 和普通编辑器直接管理内容的能力，并避免双写与同步冲突；代价是内容查询必须从文件构建索引，且内容写入不通过站点数据库完成。

核心默认不改写内容文件。唯一例外是站点所有者在 `start` / `reload` 显式传入 `--ensure-content-fields` 时：在本次内容快照构建前，仅为真缺的 `id`（UUID）与 `created`（文件 birthtime，不可用则 mtime，写为 UTC `YYYY-MM-DD`）写回磁盘；无 Front Matter 时插入完整块，非法已有值失败不覆盖。不带该参数时行为与「文件只读真相」完全一致。
