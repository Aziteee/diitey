# 站点根 `public/` 提供根镜像的站点静态资源

站点所有者需要 favicon、`robots.txt`、`/.well-known` 等路径稳定、不经构建与内容寻址的文件。核心约定站点根下固定目录 `public/`（不可配置、不可出站）；存在时按根镜像提供普通文件，目录缺失则静默关闭。请求时读磁盘、不进内容快照或有效发布视图、不钉进站点程序；仅 GET/HEAD、精确路径、无 index/列目录；解析后必须落在 `public` 真实路径树内。请求顺序上核心保留路径与构建/内容资源及主题路由优先于站点静态资源，重叠时 public 文件静默不可达。缓存使用可协商的短缓存（非 immutable），因稳定 URL 下文件可原地变更。相比把此类文件并入内容资源或主题构建，这保持内容寻址与主题信息架构的边界，代价是无原子整包切换，且被主题挡住的文件不易自发现。
