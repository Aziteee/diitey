# 站点可声明内容目录（可位于站点根之外）

站点所有者在 `site.config.ts` 用可选字段 `contentDir` 声明内容目录；省略时默认为相对站点根的 `content`。相对路径相对站点根解析，绝对路径原样使用，不展开 `~`；解析结果须为已存在的目录，并作为站点程序的一部分在启动时钉住。允许内容目录落在站点根之外（含相对路径中的 `..` 与符号链接），以便内容真相与主题/插件树分离；`sourcePath` 仍相对该内容目录，内容资源边界仍不得逃出解析后的真实内容目录。相比硬编码 `content/` 或多根扫描，这只打开「单根可配置」，代价是误配路径会在启动失败，且变更内容目录需要进程重启而非 reload。
